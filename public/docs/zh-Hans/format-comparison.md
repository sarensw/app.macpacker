## ZIP：通用压缩格式

**ZIP** 是使用最广泛的压缩格式,这是有充分理由的:它在任何地方都无需额外软件即可工作。ZIP 文件创建于 1989 年,在 macOS、Windows 和大多数 Linux 发行版上都获得原生支持。当您在 Mac 上使用内置的归档实用工具压缩文件时(右键点击 > 压缩),您就在创建 ZIP 压缩包。

### 何时使用 ZIP 压缩包

当**兼容性比压缩率更重要**时,ZIP 是您的最佳选择。如果您与可能没有专用解压工具的用户共享文件,ZIP 可以确保他们能够无障碍地打开您的压缩包。电子邮件附件、网站下载和快速文件传输都受益于 ZIP 的通用支持。

ZIP 文件使用 Deflate 压缩算法,提供中等压缩率 — 通常可将文本文档的文件大小减少 40-60%,混合内容减少 10-30%。虽然不是最高效的压缩方式,但 ZIP 通过快速的压缩和解压速度进行补偿。

### ZIP 压缩性能

现代 ZIP 实现支持 AES-256 加密以实现密码保护,使 ZIP 适用于安全文件传输。然而,macOS 归档实用工具没有公开密码保护功能 — 您需要像 MacPacker 这样的工具来[在 Mac 上对 ZIP 文件进行密码保护](./password-protect-zip)。

**ZIP 优势:**
- ✅ 在所有主流平台上无需额外软件即可工作
- ✅ 快速压缩和解压
- ✅ 无需解压整个压缩包即可访问单个文件
- ✅ 广泛的软件生态系统和成熟的工具

**ZIP 局限性:**
- ❌ 与 RAR 和 7z 相比压缩率较低
- ❌ macOS 原生支持缺少密码保护
- ❌ 对大于 4GB 的文件支持有限(需要 ZIP64 扩展)

## RAR：高压缩率与广泛支持

**RAR**(Roshal Archive)是 Eugene Roshal 于 1993 年创建的专有压缩格式。尽管是闭源的,但由于其出色的压缩率和强大的功能集,RAR 已成为分发大文件、软件和媒体集合最流行的格式之一。

### RAR 的优势与劣势

RAR 的主要优势是**固实压缩**,这是一种将多个文件作为连续数据流压缩而不是单独处理的技术。这种方法比标准 ZIP 实现 10-30% 更好的压缩,特别是对于相似文件集合(源代码、文本文档或媒体文件)。

RAR 支持多卷压缩包(将大型压缩包拆分为多个文件)、恢复记录(修复损坏的压缩包)和高级加密。这些功能使 RAR 在软件分发中很受欢迎,其中文件完整性和压缩效率至关重要。

### macOS 上的 RAR

macOS 不包含原生 RAR 支持,因此您需要第三方应用程序。MacPacker 提供无缝 RAR 解压,支持 RAR 5.0(最新版本)、密码保护的压缩包和多部分卷。在我们的完整指南中了解更多:[如何在 Mac 上解压 RAR 文件](./extract-rar)。

**RAR 优势:**
- ✅ 通过固实压缩实现出色的压缩率
- ✅ 强大的错误恢复和压缩包修复功能
- ✅ 在软件和媒体分发中广泛采用
- ✅ 强大的 AES-256 加密支持

**RAR 局限性:**
- ❌ 专有格式(创建 RAR 文件需要付费 WinRAR 许可证)
- ❌ 没有原生 macOS 或 Windows 支持(需要第三方工具)
- ❌ 与 ZIP 相比压缩和解压速度较慢

## 7-Zip (7z)：开源最大压缩率

**7z** 是由 Igor Pavlov 开发的开源压缩格式,于 1999 年首次发布。它使用先进的 LZMA 和 LZMA2 压缩算法,实现了任何广泛使用格式中最高的压缩率 — 通常比等效的 ZIP 压缩包小 30-50%。

### 为什么 7z 提供最佳压缩

7z 通过几个技术优势实现卓越的压缩。LZMA2 算法使用基于字典的压缩方案,具有大字典大小(最多 1.5GB),允许它识别和消除大数据集中的冗余。结合固实压缩(如 RAR),7z 可以实现显著的文件大小减少。

对于包含相似文件的大型压缩包 — 软件源代码存储库、文档集合或未压缩的媒体 — 7z 始终产生最小的输出文件。这使得 7z 成为分发大型开源项目、游戏模组和存档备份的首选格式,特别是在存储空间紧张的情况下。

### 7z 跨平台考虑

作为[开源](https://github.com/sarensw/macpacker)软件,7z 完全免费用于创建和解压压缩包。7-Zip 程序在 Windows 上可用,而 Mac 和 Linux 用户可以使用兼容工具如 MacPacker 或命令行实用程序(p7zip)。

然而,7z 的出色压缩有一个权衡:与 ZIP 相比压缩和解压速度较慢。创建大型 7z 压缩包可能需要相当长的时间,尽管解压性能是可以接受的。对于大多数用户来说,文件大小的节省证明了额外的处理时间是值得的。

**7z 优势:**
- ✅ 所有主流格式中最高的压缩率
- ✅ 开源格式,无许可限制
- ✅ 强大的 AES-256 加密
- ✅ 支持最大 16 EB 大小的压缩包
- ✅ 积极开发和现代功能

**7z 局限性:**
- ❌ 没有原生操作系统支持(所有平台都需要工具)
- ❌ 压缩和解压速度较慢
- ❌ 不如 ZIP 或 RAR 广泛采用

## TAR 和 GZIP：Unix 压缩标准

**TAR**(Tape Archive)和 **GZIP** 是 Unix 原生格式,分别自 1970 年代和 1980 年代以来就存在。虽然在技术上是独立的工具,但它们几乎总是一起使用来创建 `.tar.gz` 或 `.tgz` 压缩包 — 这是在类 Unix 系统(包括 macOS 和 Linux)上分发软件的标准格式。

### TAR.GZ 使用场景

TAR 将多个文件和目录捆绑到一个未压缩的压缩包中,保留 Unix 文件权限、符号链接和目录结构。然后 GZIP 使用 Deflate 算法(ZIP 使用的相同算法)压缩此 TAR 压缩包,产生中等到高的压缩率。

macOS 通过归档实用工具和命令行工具包含对 TAR 和 GZIP 的原生支持。当您下载开源软件、开发工具或服务器应用程序时,您经常会遇到 `.tar.gz` 压缩包。它们在以下场景特别常见:
- 软件源代码分发
- Unix/Linux 应用程序包
- 服务器备份和系统存档
- Python 包和 Node.js 模块

**TAR/GZIP 优势:**
- ✅ 在 macOS 和所有 Unix 系统上原生支持
- ✅ 保留 Unix 文件权限和元数据
- ✅ 开源软件的标准格式
- ✅ 对源代码和文本有良好的压缩率

**TAR/GZIP 局限性:**
- ❌ 没有内置加密(需要外部工具)
- ❌ 在 Windows 上不太常见(尽管现代工具支持)
- ❌ 必须解压整个压缩包才能访问单个文件

## DMG 和 ISO：磁盘映像格式

**DMG**(Apple Disk Image)和 **ISO**(国际标准化组织)不是传统的压缩格式 — 它们是代表整个文件系统的磁盘映像。然而,它们经常用于文件分发和备份目的,使它们与压缩格式一起值得理解。

### 何时使用磁盘映像而非压缩包

**DMG** 是 macOS 特有的,是分发 Mac 应用程序的标准格式。当您下载 Mac 软件时,您通常会收到一个 DMG 文件,它在 Finder 中作为虚拟磁盘挂载。DMG 文件可以被压缩(减少文件大小)和加密(保护内容),但它们主要是为软件安装而不是通用存档而设计的。

**ISO** 文件代表光盘映像(CD、DVD、蓝光),包含磁盘的精确扇区对扇区副本。ISO 文件不使用压缩 — 它们保留原始磁盘结构,包括可启动分区。这使得 ISO 对以下场景至关重要:
- 操作系统安装媒体(macOS、Windows、Linux)
- 需要光盘仿真的软件
- 存档 CD/DVD 收藏
- 创建可启动 USB 驱动器

MacPacker 支持 DMG 和 ISO 格式,允许您从磁盘映像中提取文件,而无需挂载它们或刻录到物理媒体。

**DMG/ISO 优势:**
- ✅ DMG 是标准的 Mac 软件分发格式
- ✅ ISO 在光盘映像方面得到通用支持
- ✅ DMG 支持压缩和加密
- ✅ 两者都保留完整的文件系统结构

**DMG/ISO 局限性:**
- ❌ DMG 仅限 macOS(与 Windows/Linux 不兼容)
- ❌ ISO 文件未压缩,可能非常大
- ❌ 不适合通用文件压缩

## 您应该使用哪种压缩格式?

选择最佳压缩格式取决于您的具体使用场景。以下是常见场景的基于证据的建议:

### 为了最大兼容性：ZIP

当与非技术用户共享文件或当您不确定接收者有哪些解压工具时,使用 ZIP。ZIP 的通用支持意味着任何人都可以打开您的文件,而无需安装额外的软件。这使得 ZIP 非常适合:
- 电子邮件附件和文件共享服务
- 网站下载和公共文件分发
- 快速个人文件压缩
- 与旧系统的兼容性

### 为了最佳压缩：7z

当文件大小是您的主要关注点且接收者愿意安装解压软件时,选择 7z。7z 实现最小的文件大小,使其非常适合:
- 带宽很重要的大型软件分发
- 长期存储的存档备份
- 通过下载分发的开源项目
- 压缩率优于便利性的情况

### 为了软件分发：RAR

RAR 在压缩效率和功能丰富性之间提供了最佳平衡。在分发大型软件包、游戏文件或媒体集合时使用 RAR,当:
- 需要多部分压缩包(拆分为多个文件)
- 恢复记录可以防止损坏
- 接收者可能有 RAR 解压工具
- 敏感内容需要强加密

### 为了 Unix 系统：TAR/GZIP

在为类 Unix 系统(macOS、Linux、BSD)分发软件或保留 Unix 文件权限至关重要时,使用 TAR.GZ。此格式是以下场景的标准:
- 开源软件源代码
- 服务器配置备份
- 开发工具和库
- 任何 Unix 元数据很重要的内容

## 常见问题解答

### 哪种压缩格式具有最佳压缩率?

7z 在所有主流压缩格式中提供最佳压缩率,通常产生比等效 ZIP 压缩包小 30-50% 的文件。这归功于先进的 LZMA2 压缩算法和固实压缩支持。RAR 以出色的压缩排在第二位,而 ZIP 提供中等压缩但速度更快。为了最大限度地减少文件大小,使用 7z;为了压缩和速度的平衡,使用 RAR;为了中等压缩的最大兼容性,使用 ZIP。

### ZIP 和 RAR 有什么区别?

ZIP 和 RAR 都是压缩格式,但由于固实压缩支持,RAR 通常实现 10-30% 更好的压缩率。ZIP 在所有主要操作系统上无需额外软件即可获得通用支持,而 RAR 需要第三方解压工具。RAR 还提供高级功能,如恢复记录和多部分压缩包支持,使其在大型软件分发中很受欢迎。ZIP 在兼容性和速度方面更好,而 RAR 在压缩效率和稳健性方面表现出色。

### 7z 比 ZIP 或 RAR 更好吗?

7z 提供比 ZIP 和 RAR 更好的压缩率(更小的文件),当文件大小是主要关注点时使其成为理想选择。然而,7z 在所有平台上都需要第三方软件(没有原生操作系统支持),并且压缩/解压速度较慢。ZIP 提供与原生操作系统支持的通用兼容性。RAR 提供了出色压缩和强大功能的中间地带。"最佳"格式取决于您的优先级:使用 7z 实现最大压缩,ZIP 实现最大兼容性,或 RAR 实现平衡功能。

### 为什么在 Mac 上使用 TAR 而不是 ZIP?

TAR(与 GZIP 结合为 `.tar.gz`)保留 Unix 文件权限、符号链接和 ZIP 可能无法正确处理的扩展文件属性。这在分发 Unix 软件、备份系统配置或在类 Unix 系统之间传输文件时很重要。macOS 基于 Unix,因此 TAR.GZ 是许多命令行工具和开源包的原生格式。对于一般文件压缩,ZIP 在 Mac 上工作良好,但 TAR.GZ 更适合技术分发和开发工作流程。

### 我可以在 macOS 上打开所有压缩格式吗?

macOS 通过内置归档实用工具原生支持 ZIP、GZIP、BZIP2 和 TAR 格式。然而,像 RAR、7z 和许多其他流行格式需要第三方应用程序。MacPacker 为 30+ 种压缩格式提供全面支持,包括 ZIP、RAR、7z、TAR、GZIP、BZIP2、DMG、ISO 等等。使用 MacPacker,您可以解压在 Mac 上遇到的任何压缩格式,而无需担心兼容性。

### 哪种格式最适合密码保护?

7z、RAR 和 ZIP 都支持 AES-256 加密用于密码保护的压缩包。7z 提供最强大的实现,能够加密文件内容和文件名(隐藏压缩包中的内容)。RAR 还提供具有额外恢复功能的强大加密。ZIP 支持 AES-256 加密,但 macOS 归档实用工具没有公开此功能 — 您需要像 MacPacker 这样的工具来创建密码保护的 ZIP 文件。为了最大安全性,使用具有强密码的 7z 或 RAR。

MacPacker 对所有格式提供同样好的支持,但对于大多数用户,我们推荐**用于共享的 ZIP**(通用兼容性)、**用于存储的 7z**(最佳压缩)和**用于 Unix 软件的 TAR.GZ**(保留元数据)。最佳格式取决于您的使用场景而不是技术优越性。由于 MacPacker 无缝处理所有格式,您不需要标准化为一种格式 — 使用您的接收者最容易打开的格式或实现您的压缩目标的格式。

### 如何在 ZIP、RAR 和 7z 之间选择?

如果您的接收者可能没有解压软件,选择 ZIP(最大兼容性)。如果文件大小至关重要且您想要最小可能的压缩包,选择 7z(最大压缩)。如果您需要多部分压缩包或恢复记录等高级功能,并且压缩效率很重要,选择 RAR(平衡功能)。对于 Mac 上的个人使用,所有三种都可以很好地与 MacPacker 配合使用 — 选择主要影响文件大小和与其他平台上接收者的兼容性。
