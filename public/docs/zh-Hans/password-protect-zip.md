# 如何在 Mac 上为 ZIP 文件设置密码保护（3 种简单方法）

macOS 不允许通过访达为 ZIP 文件设置密码保护，但您可以使用终端、MacPacker 等第三方应用或磁盘工具轻松加密 ZIP 档案。最快的方法是终端 `zip -e` 命令，它可以在几秒钟内创建 AES-256 加密的 ZIP 文件。本指南展示了三种经过验证的方法来为 Mac 上的 ZIP 文件设置密码保护，并提供每种方法的分步说明。

## 为什么 macOS 不支持密码保护的 ZIP 文件

Apple 的归档实用程序（内置于访达中）可以创建 ZIP 文件，但不支持密码保护。这是 macOS 的一个长期限制。密码保护的 ZIP 文件对于在文件传输、电子邮件附件和云存储期间保护敏感数据至关重要。常见用例包括：

- **机密文档** — 保护财务记录、合同和个人信息
- **安全文件共享** — 通过电子邮件或云存储发送加密的敏感文件
- **合规要求** — 满足处理受保护数据的安全标准
- **跨平台兼容性** — ZIP 文件在 Windows、Mac 和 Linux 上都支持密码

与 macOS 访达内置的"压缩"功能（创建未加密的 ZIP 文件）不同，以下方法添加密码保护和加密来保护您的数据。

## 方法 1：使用终端为 ZIP 文件设置密码保护（最快方法）

终端方法是在 macOS 上创建密码保护 ZIP 文件最快、最可靠的方法。它使用内置的 `zip` 命令和 AES-256 加密。

### 分步指南：使用终端创建加密 ZIP 文件

打开终端（应用程序 > 实用工具 > 终端）并导航到包含要压缩文件的文件夹：

```bash
cd /path/to/your/files
```

要创建带密码保护的加密 ZIP 文件：

```bash
zip -er archive.zip folder_name
```

**标志说明：**
- `-e` — 启用密码加密（系统会提示您输入密码）
- `-r` — 递归包含所有文件和子文件夹

运行命令后，终端会提示您输入并验证密码：

```bash
Enter password:
Verify password:
```

**重要提示：** 输入时您的密码不会显示（出于安全考虑这是正常的）。

### 不同场景的终端命令

**加密特定文件（而不是文件夹）：**

```bash
zip -e documents.zip file1.pdf file2.docx file3.xlsx
```

**加密单个文件：**

```bash
zip -e confidential.zip sensitive-document.pdf
```

**使用最大压缩加密：**

```bash
zip -9er archive.zip folder_name
```

`-9` 标志应用最大压缩（速度较慢但创建更小的文件）。

**从加密中排除文件：**

```bash
zip -er archive.zip folder_name -x "*.tmp" "*.log"
```

这会创建一个加密档案，但排除临时文件和日志。

### ZIP 密码的安全最佳实践

创建密码保护的 ZIP 文件时，请遵循以下安全准则：

1. **使用强密码** — 至少 12 个字符，包含大写、小写、数字和符号
2. **切勿通过同一渠道共享密码** — 分别发送 ZIP 文件和密码（例如，通过电子邮件发送文件，通过短信发送密码）
3. **避免使用词典单词** — 使用随机密码生成器或密码短语
4. **考虑加密强度** — 终端 `zip -e` 命令使用 AES-256 加密，非常安全
5. **测试您的档案** — 在发送之前始终测试提取档案，以验证密码是否有效

## 方法 3：使用磁盘工具创建加密 DMG（替代方法）

如果您需要 macOS 原生 GUI 解决方案进行文件加密，磁盘工具可以创建加密的 DMG（磁盘映像）文件。虽然 DMG 文件不是 ZIP 档案，但它们提供强大的加密，并在 Apple 生态系统中运行良好。

### 分步指南：创建加密 DMG

1. 打开**磁盘工具**（应用程序 > 实用工具 > 磁盘工具）
2. 前往**文件 > 新建映像 > 来自文件夹的映像...**
3. 选择要加密的文件夹
4. 选择**加密：** 选择"128 位 AES 加密"或"256 位 AES 加密"
5. 输入并验证您的密码
6. 将**映像格式：** 设置为"压缩"以节省磁盘空间
7. 点击**保存**

磁盘工具将创建一个需要密码才能装载和访问的加密 `.dmg` 文件。

### DMG 与 ZIP 文件加密对比

**何时使用加密 DMG：**
- 仅与其他 Mac 用户共享文件
- 在外部驱动器上存储加密备份
- 需要原生 macOS 集成（双击装载）
- 更喜欢使用 GUI 工具而不是终端命令

**何时使用密码保护的 ZIP：**
- 与 Windows 或 Linux 用户共享文件
- 电子邮件附件（ZIP 更普遍认可）
- 需要跨平台兼容性
- 更小的文件大小至关重要

为了在所有操作系统中实现最大兼容性，密码保护的 ZIP 文件（方法 1）是更好的选择。

## AES-256 与旧版 ZIP 加密：哪个更安全？

并非所有 ZIP 加密都是相同的。了解加密类型有助于您选择正确的安全级别。

### 旧版 ZIP 2.0 加密（弱）

较旧的 ZIP 工具使用旧版 ZIP 2.0 加密，也称为"ZipCrypto"。这种加密方法**不安全**，可以使用现成的工具破解。

**特征：**
- 快速加密/解密
- 广泛兼容（即使非常旧的软件也可以打开）
- 容易受到已知明文攻击
- 不推荐用于敏感数据

### AES-256 加密（强）

现代 ZIP 工具，包括 macOS 终端的 `zip -e` 命令，使用 **AES-256 加密**（具有 256 位密钥的高级加密标准）。

**特征：**
- 军用级加密强度
- 使用强密码可抵御暴力攻击
- 受大多数现代档案工具支持（WinZip、7-Zip、MacPacker 等）
- 推荐用于所有敏感数据

**验证：** macOS 终端在运行 `zip -e` 时默认使用 AES-256。您可以通过检查 ZIP 文件属性或使用显示加密类型的工具测试提取来验证这一点。

## 常见问题故障排除

### 提取时"密码不正确"

如果您确定密码正确：

1. **检查拼写错误** — 密码区分大小写
2. **检查键盘布局** — 确保大写锁定已关闭并选择了正确的语言输入
3. **尝试复制/粘贴密码** — 消除输入错误
4. **验证加密兼容性** — 某些旧工具不支持 AES-256；尝试使用 MacPacker 或现代工具提取

### 创建的 ZIP 文件没有密码保护

如果 `zip -e` 不提示输入密码：

1. **验证 `-e` 标志** — 它必须在输出文件名之前：`zip -e archive.zip files`
2. **检查终端输出** — 查找"Enter password:"提示
3. **测试档案** — 尝试提取它；如果不需要密码，则未加密
4. **使用显式加密** — 某些系统需要 `zip -er`（带递归标志）才能正确启用加密

### 无法记住密码

**不幸的是，没有办法恢复忘记的 ZIP 密码。** ZIP 加密（尤其是 AES-256）旨在在没有正确密码的情况下无法破解。

**预防策略：**
- 使用密码管理器（1Password、Bitwarden 等）存储 ZIP 密码
- 保留与 ZIP 文件分开的密码安全记录
- 与可信收件人共享时使用密码提示
- 对于关键档案，将密码存储在安全保险库中

### Windows 用户无法打开 Mac 创建的 ZIP 文件

这种情况很少见，但在非常旧的 Windows 系统上可能发生：

1. **确保 AES-256 支持** — Windows 10+ 原生支持 AES-256；较旧版本可能需要 WinZip 或 7-Zip
2. **使用 7-Zip 测试** — 具有完整 AES-256 支持的免费 Windows 工具：[www.7-zip.org](https://www.7-zip.org)
3. **验证文件传输完整性** — 确保 ZIP 文件在下载过程中没有损坏
4. **检查文件名兼容性** — 避免 Windows 不支持的文件名中的特殊字符

## 常见问题解答

### macOS 访达可以创建密码保护的 ZIP 文件吗？

不可以，macOS 访达内置的"压缩"选项只创建没有密码保护或加密的标准 ZIP 文件。您必须使用终端、第三方应用程序或磁盘工具加密 DMG 文件来添加密码保护。

### 在 Mac 上为 ZIP 文件设置密码保护的最安全方法是什么？

使用 `zip -e` 命令的终端方法，它应用 AES-256 加密（军用级安全性）。将此与强密码（12+ 个字符，混合大小写、数字、符号）结合使用，并且切勿通过与 ZIP 文件相同的渠道共享密码。

### 如何在 Mac 上打开密码保护的 ZIP 文件？

macOS 归档实用程序可以提取密码保护的 ZIP 文件 — 只需双击文件并在提示时输入密码。或者，使用 MacPacker 获得更好的提取体验，支持损坏的档案、多部分文件和预览功能。

### 我可以为现有 ZIP 文件添加密码保护吗？

不可以，您无法向现有 ZIP 文件添加密码。您必须提取内容并使用上述方法之一创建新的密码保护档案（推荐使用终端 `zip -e` 命令）。

### macOS 对密码保护的 ZIP 文件使用什么加密？

macOS 终端 `zip -e` 命令默认使用 **AES-256 加密**，这与政府和军事组织使用的加密标准相同。与强密码结合使用时，这提供了针对暴力攻击的强大保护。

### 如何批量创建多个密码保护的 ZIP 文件？

使用终端脚本自动为多个文件夹创建 ZIP：

```bash
for folder in */; do
    zip -er "${folder%/}.zip" "$folder"
done
```

此脚本循环遍历当前目录中的所有文件夹，并为每个文件夹创建加密的 ZIP。系统会提示您为每个档案输入密码。

### 密码保护的 ZIP 文件有文件大小限制吗？

标准 ZIP 文件支持每个文件最多 4 GB，每个档案最多 65,535 个文件。对于更大的文件，请使用 **ZIP64 格式**（在需要时由现代 `zip` 工具自动启用），它支持几乎无限的文件大小。macOS 上的终端 `zip` 命令自动处理 ZIP64。
